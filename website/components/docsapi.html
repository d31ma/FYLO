NoSQL API

The collection API stays straightforward: create a collection, write JSON documents, query by indexed fields, then patch or delete by document id.

createCollection()
Create a collection directory and FYLO metadata layout.
putData()
Insert a document and receive a generated TTID.
getDoc()
Read one document and optionally keep listening for changes.
findDocs()
Filter many documents with async iteration and collection helpers.
patchDoc() / patchDocs()
Update one document by id or patch many by query.
delDoc() / delDocs()
Delete one document or a query-selected set.
CRUD
const fylo = new Fylo({ root: '/mnt/fylo' }) const userId = await fylo.putData('users', { name: 'Jane Doe', age: 29, team: 'platform' }) const user = await fylo.getDoc('users', userId).once() for await (const doc of fylo.findDocs('users', { $ops: [{ age: { $gte: 18 } }] }).collect()) { console.log(doc) }

SQL API

FYLO also accepts SQL-like commands through executeSQL(). That is useful for app-facing reads, inserts, updates, deletes, and collection creation without swapping to a separate database process.

SQL
await fylo.executeSQL('CREATE TABLE posts') await fylo.executeSQL( 'INSERT INTO posts VALUES { "title": "Hello", "published": true }' ) const posts = await fylo.executeSQL( 'SELECT * FROM posts WHERE published = true' )

Query Operators

The typed query surface supports common comparison and filter operations across object fields, timestamps, and join conditions.

$eq $ne $gt $lt $gte $lte $like $contains
Operator Required Details
$eq / $neNoExact matches and exclusions for scalar fields.
$gt / $lt / $gte / $lteNoRange filtering for numeric fields and timestamps.
$likeNoString matching for query-style search clauses.
$containsNoContains-style checks for arrays and similar shapes.
$created / $updatedNoTimestamp range filters at the query level.

Realtime Listeners

FYLO keeps an append-only event journal per collection. Listener APIs read from that journal so your code can watch a single document or a matching result set over time.

Listeners
for await (const doc of fylo.findDocs('users', { $ops: [{ role: { $eq: 'admin' } }] })) { console.log(doc) } for await (const deletedId of fylo.getDoc('users', userId).onDelete()) { console.log('deleted', deletedId) }

Joins

Cross-collection joins still exist in the package surface. You can call joinDocs() directly or express joins through SQL.

JoinDocs
const joined = await fylo.joinDocs({ $leftCollection: 'posts', $rightCollection: 'users', $mode: 'inner', $on: { userId: { $eq: 'id' } } })

Bulk Import / Export

FYLO still exposes programmatic bulk helpers for importing JSON from a URL and streaming documents back out. Imports now default to a safer network posture for server-side fetches.

Bulk Data
const count = await fylo.importBulkData( 'users', new URL('https://data.example.com/users.json'), { limit: 1000, maxBytes: 5 * 1024 * 1024, allowedHosts: ['data.example.com'] } ) for await (const doc of fylo.exportBulkData('users')) { console.log(doc) }
Import guardrails. importBulkData() expects JSON, caps responses at 50 MiB by default, and rejects localhost, loopback, link-local, private-network, and private-address HTTP(S) imports unless you explicitly opt in.
Trusted internal URLs. Use allowedHosts, maxBytes, and limit for normal imports. Only set allowPrivateNetwork: true for sources your app fully controls.

Sync Hooks

FYLO does not ship its own cloud sync engine. Instead, it lets you attach hooks that run after local writes and deletes so your application can replicate the filesystem root however it wants.

Hooks
const fylo = new Fylo({ root: '/mnt/fylo', syncMode: 'await-sync', sync: { async onWrite(event) { const file = Bun.file(event.path) await myS3Client.putObject({ key: event.collection + '/' + event.docId + '.json', body: await file.arrayBuffer() }) }, async onDelete(event) { await myS3Client.deleteObject({ key: event.collection + '/' + event.docId + '.json' }) } } })
Mode Required Details
await-syncNoDefault. Waits for the hook and throws if replication fails.
fire-and-forgetNoCommits locally first and runs your hook in the background.
Error shape. When an awaited sync fails after the local write succeeds, FYLO throws FyloSyncError with the collection, doc id, path, and operation.

CLI

The current package publishes fylo.query for running SQL-style commands from the shell.

Shell
FYLO_ROOT=/tmp/fylo fylo.query "CREATE TABLE users" FYLO_ROOT=/tmp/fylo fylo.query "SELECT * FROM users"

Compatibility Notes

FYLO keeps a few older method names for compatibility, but the current runtime no longer centers queued writes, Redis workers, or built-in migration commands.

Current direction. rollback() is now a no-op, and legacy queue helpers such as getJobStatus(), getQueueStats(), replayDeadLetter(), and processQueuedWrites() throw an error that explains the feature was removed.

If you run across older Redis or migration references in historical discussions, treat them as previous design stages rather than the public shape of @delma/fylo@2.2.0.
