@d31ma/fylo

Filesystem-first document storage for any language.

FYLO keeps one canonical file per document and one collection index file for fast queries. The filesystem is the source of truth. Indexes are accelerators — rebuildable from documents at any time.

A Bun-native library plus a standalone executable binary and HTTP gateway, so any language can use it. Two runtimes: a server engine for filesystem-backed storage and a browser engine for OPFS-backed local-first apps. No native addons, no separate database processes.

Mental model. Document files are authoritative. Index files can be rebuilt from those documents if they ever drift.
Filesystem Layout
<root>/<collection>/ .fylo/ docs/ 4U/ 4UUB32VGUDW.json indexes/ <collection>.idx.json events/ <collection>.ndjson

Installation

Install the published library from npm:

Bash
bun add @d31ma/fylo

A minimal setup can be as small as a root path plus one collection:

TypeScript
import Fylo from '@d31ma/fylo' const fylo = new Fylo('/mnt/fylo') await fylo.users.create() const id = await fylo.users.put({! '\x7b' } name: 'Ada', role: 'admin', tags: ['engineering', 'platform'] {! '\x7d' }) const doc = await fylo.users.get(id).once()

Storage Model

Configure a root with FYLO_ROOT or pass root to the constructor. If neither is set, FYLO defaults to <current working directory>/.fylo-data.

Configuration
# Set the filesystem root FYLO_ROOT=/mnt/fylo # Constructor alternative const fylo = new Fylo('/mnt/fylo')
Compatibility alias. For older filesystem-mode experiments, FYLO still accepts s3FilesRoot for backward compatibility. The preferred modern name is root or FYLO_ROOT.

Use AWS S3 Files or another replication layer outside the package. FYLO owns local document writes, query behavior, index maintenance, and realtime journaling. You own how that root gets mirrored elsewhere.

Environment Variables

Variable Required Details
FYLO_ROOT Recommended Filesystem root for collections. Defaults to .fylo-data in the current working directory.
FYLO_SCHEMA Optional Directory containing collection JSON schemas for validation and encryption metadata.
FYLO_STRICT Optional When truthy, validates writes with @d31ma/chex.
FYLO_ENCRYPTION_KEY Conditional Required when any schema declares $encrypted fields.
FYLO_CIPHER_SALT Recommended Unique per deployment salt for PBKDF2 key derivation.
FYLO_LOGGING Optional Enables extra logging during bulk import and related flows.
Default root. If no root is configured, FYLO stores data under .fylo-data in the current working directory.
