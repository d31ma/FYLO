NoSQL API

The collection API stays straightforward: create a collection, write JSON documents, query by indexed fields, then patch or delete by document id.

create()
Create a collection directory and FYLO metadata layout.
put() / put.batch()
Insert one document, or many at once, each with a generated TTID.
get()
Read one document and optionally keep listening for changes.
find() / find.deleted()
Iterate live documents, or list soft-deleted ones, with collection helpers.
patch() / patch.many()
Update one document by id or patch many by query.
delete() / delete.many()
Delete one document or a query-selected set.
CRUD
const fylo = new Fylo('/mnt/fylo') const userId = await fylo.users.put({! '\x7b' } name: 'Jane Doe', age: 29, team: 'platform' {! '\x7d' }) const user = await fylo.users.get(userId).once() for await (const doc of fylo.users.find({! '\x7b' } $ops: [{! '\x7b' } age: {! '\x7b' } $gte: 18 {! '\x7d' } {! '\x7d' }] {! '\x7d' }).collect()) {! '\x7b' } console.log(doc) {! '\x7d' }

SQL API

FYLO also accepts SQL-like commands through sql(). That is useful for app-facing reads, inserts, updates, deletes, and collection creation without swapping to a separate database process.

SQL
await fylo.posts.create() await fylo.sql`INSERT INTO posts VALUES {! '\x7b' } "title": "Hello", "published": true {! '\x7d' }` const posts = await fylo.sql`SELECT * FROM posts WHERE published = true`

Query Operators

The typed query surface supports common comparison and filter operations across object fields, timestamps, and join conditions.

$eq $ne $gt $lt $gte $lte $like $contains
Operator Required Details
$eq / $neNoExact matches and exclusions for scalar fields.
$gt / $lt / $gte / $lteNoRange filtering for numeric fields and timestamps.
$likeNoString matching for query-style search clauses.
$containsNoContains-style checks for arrays and similar shapes.
$created / $updatedNoTimestamp range filters at the query level.

Realtime Listeners

FYLO keeps an append-only event journal per collection. Listener APIs read from that journal so your code can watch a single document or a matching result set over time.

Listeners
for await (const doc of fylo.users.find({! '\x7b' } $ops: [{! '\x7b' } role: {! '\x7b' } $eq: 'admin' {! '\x7d' } {! '\x7d' }] {! '\x7d' })) {! '\x7b' } console.log(doc) {! '\x7d' } for await (const deletedId of fylo.users.get(userId).onDelete()) {! '\x7b' } console.log('deleted', deletedId) {! '\x7d' }

Joins

Cross-collection joins still exist in the package surface. You can call join() directly or express joins through SQL.

Join
const joined = await fylo.join({! '\x7b' } $leftCollection: 'posts', $rightCollection: 'users', $mode: 'inner', $on: {! '\x7b' } userId: {! '\x7b' } $eq: 'id' {! '\x7d' } {! '\x7d' } {! '\x7d' })

Bulk Import / Export

FYLO still exposes programmatic bulk helpers for importing JSON from a URL and streaming documents back out. Imports now default to a safer network posture for server-side fetches.

Bulk Data
const count = await fylo.users.import( new URL('https://data.example.com/users.json'), {! '\x7b' } limit: 1000, maxBytes: 5 * 1024 * 1024, allowedHosts: ['data.example.com'] {! '\x7d' } ) for await (const doc of fylo.users.export()) {! '\x7b' } console.log(doc) {! '\x7d' }
Import guardrails. importBulkData() expects JSON, caps responses at 50 MiB by default, and rejects localhost, loopback, link-local, private-network, and private-address HTTP(S) imports unless you explicitly opt in.
Trusted internal URLs. Use allowedHosts, maxBytes, and limit for normal imports. Only set allowPrivateNetwork: true for sources your app fully controls.

Sync Hooks

FYLO does not ship its own cloud sync engine. Instead, it lets you attach hooks that run after local writes and deletes so your application can replicate the filesystem root however it wants.

Hooks
const fylo = new Fylo('/mnt/fylo', {! '\x7b' } syncMode: 'await-sync', sync: {! '\x7b' } async onWrite(event) {! '\x7b' } const file = Bun.file(event.path) await myS3Client.putObject({! '\x7b' } key: event.collection + '/' + event.docId + '.json', body: await file.arrayBuffer() {! '\x7d' }) {! '\x7d' }, async onDelete(event) {! '\x7b' } await myS3Client.deleteObject({! '\x7b' } key: event.collection + '/' + event.docId + '.json' {! '\x7d' }) {! '\x7d' } {! '\x7d' } {! '\x7d' })
Mode Required Details
await-syncNoDefault. Waits for the hook and throws if replication fails.
fire-and-forgetNoCommits locally first and runs your hook in the background.
Error shape. When an awaited sync fails after the local write succeeds, FYLO throws FyloSyncError with the collection, doc id, path, and operation.

HTTP Gateway

Embed a REST API server directly in a JS app with serveFyloHttp, or use the CLI to stand one up so any language can call FYLO over HTTP:

Server
import {! '\x7b' } serveFyloHttp {! '\x7d' } from '@d31ma/fylo/server' serveFyloHttp({! '\x7b' } root: '/mnt/fylo', port: 3000, token: 'my-secret-token' {! '\x7d' })

The gateway exposes a full REST surface: GET /v1/:collection with URL filter parsing, POST /v1/sql for SQL queries, POST /v1/exec for multi-statement execution, and OpenAPI spec at /v1/openapi.json.

URL filters. Query via query-string operators: ?field.eq=value, ?field.gte=10, ?field.like=pattern, ?field.contains=item.

Compatibility Notes

FYLO keeps a few older method names for compatibility, but the current runtime no longer centers queued writes, Redis workers, or built-in migration commands.

Current direction. rollback() is now a no-op, and legacy queue helpers such as getJobStatus(), getQueueStats(), replayDeadLetter(), and processQueuedWrites() throw an error that explains the feature was removed.

If you run across older Redis or migration references in historical discussions, treat them as previous design stages rather than the current public API.
